# Database Schema (MVP)

The MVP PostgreSQL schema contains the following tables:

1. `events`
- Stores normalized inbound security telemetry.
- Key fields: `source`, `event_type`, `payload`, `occurred_at`, `correlation_id`.

2. `alerts`
- Stores triaged alert records linked to events.
- Key fields: `event_id`, `severity`, `title`, `description`, `status`, `metadata`.

3. `attack_runs`
- Stores replay/simulation run state.
- Key fields: `name`, `status`, `started_at`, `finished_at`.

4. `findings`
- Stores findings generated during a run.
- Key fields: `attack_run_id`, `event_id`, `alert_id`, `finding_type`, `detail`, `score`.

5. `coverage_evaluations`
- Stores per-run coverage evaluation outputs.
- Key fields: `attack_run_id`, `ruleset_id`, `scenario`, `result`, `score`, `evaluated_at`.

6. `rule_proposals`
- Stores proposed detection logic generated by triage/analysis.
- Key fields: `source_event_id`, `title`, `rationale`, `query`, `status`.

7. `replay_validations`
- Stores validation status of replay outputs.
- Key fields: `attack_run_id`, `status`, `details`, `validated_at`.

8. `rulesets`
- Stores versioned rule bundles.
- Key fields: `name`, `version`, `description`, `is_active`, `rules`.

9. `coverage_snapshots`
- Stores periodic rolled-up coverage snapshots.
- Key fields: `ruleset_id`, `coverage_percent`, `summary`, `snapshot_at`.
